The Basics

CSS stands for Cascading Style Sheets. It allows you to style your web-pages in very specific ways. Primarily, you can style your web-page in three ways using CSS syntax (In-line, Embedded, and External) (1)In-line styling: 

<h1 color:red; font-weight:normal;>This text will be affected by both of those properties </h1>

(2)Embedded styling using the Style element (this will make a global h1 selector):

<head>
  <style>
     h1 {
      color:red;
      font-weight:normal;
     }
  </style>
</head>
<body>
  <h1>This text will be affected by both of those properties</h1>
</body>

(3)Externalized styling using the link element. The same syntax that we used above, as far as CSS syntax, also applies here.First, you need to create a normal text document (the same as you did for HTML). Then, copy all of your CSS code into it. When you go to save it, save it as a .css file. Then, in the HTML file that you want to link to, place this line of code in the head:

<head>
  <link href="yourCssFileName.css" rel="stylesheet" type="text/css">
</head>

Using Classes

You can also use CSS to make non-global selectors. Meaning, only elements with their class set with the same value as the class in the head will be affected. This also means that you can apply this class to any element.

<head>
  <style>
     .alert {
      color:red;
      font-weight:normal;
     }
  </style>
</head>
<body>
  <h1 class="alert">This text will be affected by both of those properties</h1>
  <h1 class="alert">This text will also be affected by both of those properties</h1>
</body>

This will make a non-global selector that will only apply to h2s with their classes set to alert:

<head>
  <style>
     h2.alert {
      color:red;
      font-weight:normal;
     }
  </style>
</head>
<body>
  <h2 class="alert">This text will be affected by both of those properties</h2>
  <h2>This text will not be affected by both of those properties</h2>
</body>

Dealing with Fonts

There is a very big list of things CSS can style in the fonts category. A few of them are: font-size, font-family, font-weight, font-style.

Font-Size

When dealing with font sizes, 2em means that whatever the default font size is ie(16px) it will double that ie(32px). If we said font-size:3em;, it would triple it ie(48px). This allows us to keep a united, global look across all of the platforms our site might be viewed on.

Font-Family

A very useful resource for finding font families, is the Google Fonts site. When you find a font that you like, copy the link and place it in the head of the page that you want to apply it to. It is recommended that you place it before the link to the style-sheet that you're using. After that, copy the CSS code in the Specify in CSS section for that font. Then, in you your CSS style-sheet, paste that code in the body tag (if you want to apply the font to the entire page).

Font-Weight

The font weight has to do with the boldness of the text. The thickness of the font can range between 100 and 900 with 100 being the lightest and 900 being the darkest. In your style-sheet, it will look like this:

h1 {
 font-weight: 100;
}

You can also set the font-weight to normal witch is equal to 400. And, the regular bold tag is equal to 700.

Font-Style

Font style is used to add or remove the italic style. It has three different styles: Italic, Oblique, and Normal.

More CSS Properties

Here, I'm just going to list a few more random CSS properties that may, one day, come in handy:

  • text-align
  • line-height (line height affects the height of the space in between all of the text, like in a paragraph. Usually, you want the line to be bigger than the font size. ie : font-size: 1.5; line-height: 2.5; If you use a relative unit as the line height, it will always look at the font-size and then change based on that. The relative units are: em and %. ie : font-size: 20; line-height: 150%; Because the line-height is set to 150%, it will always be 1 1/2 times as big as the nearest unit in this case the font-size property)
  • margin:top left/right bottom
  • *width: %60
  • font:%100 (in this case, the %100 means about the same as 1em. It just means "give me your default font size".)
  • background-color (or, you could just do background: and specify what kind of background you'd like)
  • color (this color property acts as a "foreground" property. If you use it with a h element, it will affect it's colour.)
  • border-bottom: 10px solid gold; (this boarder can be set to be right, left, top, or bottom.)
  • text-decoration: (there are four properties that can go here underline, over-line, strike-through, and none)
*The width property is really neat, see if you can mess around with it a bit.

Using the display property

There are several different properties that we can use with this. The display property deals mainly with block and in-line elements. It will allow you to set an in-line element, such as span, to act like a block element. And, it allows a block element, such as a header tag, to act like an in-line element. To see more information about block elements and in-line elements, please see the Block vs. In-line section on the Home page. For more detailed info about the display property, please see this MDN page on the CSS display property. The three main properties, are in-line, block, and in-line-block. If set as a in-line-block, the element will act like an in-line element, but you will be able to edit the size that it takes up, like you can with a block element. The margins will also be set both vertically and horizontally if you set the element as a in-line-block. Otherwise, an in-line element's margins properties will only be set horizontally
