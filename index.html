WAI-ARIA stands for Web Accessibility Initiative – Accessible Rich Internet Applications. A large aspect of WAI-ARIA is that it allows us to add roles to our structure tags: Nav, Article, Section and Aside. Main, Header, and Footer. This is done by creating properties for those tags by using these roles:Application, Banner, Complementary, Contentinfo, Form, Main, Navigation, and Search It would look like this: <Nav role="navigation"></Nav>

Block vs. Inline

Block Elements

A block level element will be equal in height to whatever is in between the start and end tags of that element. Their width, however, spans the entire width of the webpage. That's why all block elements are stacked vertically. They can be nested inside each other, and can wrap around in-line elements.

Inline Elements

An inline element will be equal to both the height and width of whatever is in between the start and end tags of the element. They can nest other inline elements, but can't nest block elements. The only exception to this rule, is the anchor tag witch can nest block elements within it. <span></span>, <strong></strong>, and <a></a> tags are all examples of inline elements.

Using the Display Element

You can change the default nature of a tag from inline to block or the other way around, using a CSS property called display. For more information, see the Using the display property section on the CSS page.

Lists

Lists can be very helpful when you're trying to display a group of information that belongs together. There are several different types of lists. The three that I use are called Unordered, Ordered, and Descriptive. Click the button below to view an example of each.

List Example

This is an example of an unordered list:

<ul>
  <li>Item #1</li>
  <li>Item #2</li>
  <li>Item #3</li>
</ul>

This is an example of an ordered list:

<ol>
  <li>Item #1</li>
  <li>Item #2</li>
  <li>Item #3</li>
</ol>

This is an example of an descriptive list:

<dl>
  <dt>Item #1</dt>
  <dd>Description of Item #1</dd>
  <dt>Item #2</dt>
  <dd>Description of Item #2</dd>
</dl>
